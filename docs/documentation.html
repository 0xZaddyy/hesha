Getting Started

Quick Start

# Clone the repository
git clone https://github.com/heshaorg/hesha.git
cd hesha

# Build all components
cargo build --release

# Run tests
cargo test

# Start an issuer node
cargo run -p issuer-node

Core Concepts

  • Proxy Numbers: Privacy-preserving phone numbers in the format +[country_code]00[digits] (e.g., +10012345678, +440012345678)
  • Attestations: Cryptographically signed JWT tokens proving phone ownership
  • Issuers: Services that verify real phone numbers and issue attestations
  • Binding Proofs: Cryptographic mechanism preventing proxy number substitution

Specifications

Protocol Flow

  1. Phone Verification: User proves ownership via SMS/call to issuer
  2. Proxy Generation: Deterministic generation based on phone hash and user key
  3. Attestation Creation: Issuer signs JWT with proxy number and binding proof
  4. Offline Verification: Services verify attestations using issuer's public key

Attestation Format

{
  "proxy_number": "+10012345678",
  "phone_hash": "sha256:hex_hash",
  "user_pubkey": "base64_ed25519_pubkey",
  "binding_proof": "sig:base64_signature",
  "iss": "issuer.example.com",
  "exp": 1735689600,
  "iat": 1704067200,
  "jti": "unique_jwt_id",
  "nonce": "hex_nonce",
  "salt": "base64_salt"
}

Implementation Guide

For Service Providers

Integrate Hesha to accept proxy numbers instead of real phone numbers:

use hesha_core::{parse_attestation, verify_attestation};
use hesha_types::ProxyNumber;

// Accept a proxy number and attestation
let proxy = ProxyNumber::new("+10012345678")?;
let attestation_jwt = "eyJ..."; // User provides this

// Verify the attestation
let result = verify_attestation(attestation_jwt).await?;
if result.attestation.proxy_number == proxy {
    // Valid attestation for this proxy number
    println!("Verified phone ownership without seeing real number!");
}

For Issuers

Run an issuer node to verify phones and issue attestations:

// Configure issuer node
let config = IssuerConfig {
    domain: "issuer.example.com",
    private_key_path: "keys/issuer.key",
    verification_backend: SmsBackend::Twilio(twilio_config),
};

// Run the issuer service
issuer_node::run(config).await?;

For Wallet Developers

Build applications that manage user attestations:

use hesha_client::HeshaClient;

// Request attestation from issuer
let client = HeshaClient::new("https://issuer.example.com");
let attestation = client
    .request_attestation(phone_number, user_keypair)
    .await?;

// Store attestation securely
wallet.store_attestation(attestation)?;

API Reference

Issuer API Endpoints

Endpoint Method Description
/verify-phone POST Initiate phone verification via SMS/call
/attest POST Complete verification and receive attestation
/.well-known/hesha/pubkey.json GET Retrieve issuer's public key for verification

Security Considerations

Cryptographic Design

  • Ed25519 for all digital signatures
  • SHA-256 for phone number hashing
  • JWT format with standard validation
  • UUID v4 for replay protection

Security Properties

  • ‚úÖ Phone numbers never exposed
  • ‚úÖ Attestations unforgeable without issuer key
  • ‚úÖ Binding proofs prevent substitution attacks
  • ‚úÖ Offline verification prevents tracking
  • ‚úÖ User consent required for each use
